# Guest Wiring

How crates are wired into the WASM guest project (`$PROJECT_DIR/src/lib.rs`). This section only applies when `$PROJECT_DIR/src/lib.rs` exists.

For guest export patterns, handler signatures, and code examples, see [guest-patterns.md](guest-patterns.md).

## Guest Architecture

Domain crates (generated by this skill) contain business logic. The guest wires them together via the `Provider` and the guest interfaces and exposes them to the runtime.

### Guest Interfaces

The WASM guest is the top-level project that compiles to a `.wasm` component. It exports one or more of HTTP, messaging, and WebSocket WASI guest interfaces via their WASI export interfaces:

- HTTP: `wasip3::exports::http::handler::Guest` -- see [HTTP Handler Setup](guest-patterns.md#http-handler-setup)
- Messaging: `omnia_wasi_messaging::incoming_handler::Guest` -- see [Message Handler Setup](guest-patterns.md#message-handler-setup)
- WebSocket: `omnia_wasi_websocket::incoming_handler::Guest` -- see [WebSocket Handler Setup](guest-patterns.md#websocket-handler-setup)

The interfaces are used by the runtime to invoke the guest and forward requests to the domain crates. The guest may export multiple interfaces; one of each type.

### Capability Provider

The capability `Provider` implements capability traits needed by all domain crates. It is the means by which they access WASI runtime capabilities.

## Capability Provider

The capability `Provider` implements capability traits required by domain crates. It is the means by which they access WASI runtime capabilities.

Unless otherwise specified, the capability `Provider` uses each trait's default implementation. While this can be manually overridden, it is outside the scope of this skill.

```rust
#[derive(Clone, Default)]
pub struct Provider;

impl Provider {
    #[must_use]
    pub fn new() -> Self {
        Self
    }
}

impl Config for Provider {}
impl HttpRequest for Provider {}
impl Identity for Provider {}
impl Publish for Provider {}
impl StateStore for Provider {}
```

## Injecting a New Crate

When adding a generated crate to an existing guest:

### 1. Add Cargo.toml Dependency

```toml
[dependencies]
my-crate = { path = "crates/my-crate" }
```

### 2. Add Imports

```rust
use my_crate::{MyRequest, MyResponse};
```

### 3. Add Routes (HTTP), Topic Arms (Messaging), or WebSocket Events (WebSocket)

```rust
// HTTP -- add route to the Axum router:
.route("/my-endpoint", post(my_handler))

// Messaging -- add topic arm to the messaging dispatcher:
t if t.contains("my-topic.v1") => my_topic_handler(message.data()).await,

// WebSocket -- add handler delegation inside the existing WebSocket Guest impl:
handle_my_event(event.data()).await
    .map_err(|e| WsError::Other(e.to_string()))
```

If the guest does not yet have a WebSocket Guest, add the full export block:

```rust
use omnia_wasi_websocket::types::{Error as WsError, Event};

struct WebSocketGuest;
omnia_wasi_websocket::export!(WebSocketGuest);

impl omnia_wasi_websocket::incoming_handler::Guest for WebSocketGuest {
    #[omnia_wasi_otel::instrument(name = "websocket_guest_handle")]
    async fn handle(event: Event) -> Result<(), WsError> {
        handle_my_event(event.data()).await
            .map_err(|e| WsError::Other(e.to_string()))
    }
}
```

### 4. Add Handler Functions

One function per route, topic, or WebSocket event, following the patterns in [guest-patterns.md](guest-patterns.md).

### 5. Update Provider (if needed)

If the new crate uses a capability trait not yet implemented on the Provider:

```rust
impl Publish for Provider {}     // Add if crate uses Publish
impl Identity for Provider {}    // Add if crate uses Identity
impl Broadcast for Provider {}   // Add if crate uses Broadcast
impl TableStore for Provider {}  // Add if crate uses TableStore
```

### Rules

- **Append only** -- do not replace or reorder existing routes/topics/WebSocket handlers
- **No duplicates** -- check before adding routes, topics, WebSocket handlers, imports, dependencies
- **All handler functions** get `#[omnia_wasi_otel::instrument]`
- **Add `ensure_env!` entries** for any new config keys
