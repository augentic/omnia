# Project Structure

Directory layout for a WASM guest project.

## Directory Tree

```text
<guest-name>/
├── .cargo/
│   └── config.toml          # Registry + credential providers
├── .github/
│   └── workflows/
│       ├── audit.yaml       # Daily security audit
│       ├── ci.yaml          # CI on every push
│       ├── patch.yaml       # Create patch release
│       ├── publish.yaml     # CI → Publish → Deploy pipeline
│       └── release.yaml     # Create release
├── .vscode/
│   └── settings.json        # rust-analyzer wasm32 target
├── crates/                   # (optional) Local domain crates
├── examples/
│   ├── <guest-name>.rs      # Runtime entry point
│   └── .env.example         # Environment template
├── src/
│   └── lib.rs               # Guest module (HTTP + Messaging)
├── supply-chain/            # cargo-vet files
├── Cargo.toml
├── clippy.toml
├── deny.toml
├── Makefile.toml
├── rust-toolchain.toml
└── rustfmt.toml
```

## Key Files

| File | Purpose |
| ---- | ------- |
| `src/lib.rs` | HTTP handler, Messaging handler, WebSocket handler, Provider |
| `examples/<guest-name>.rs` | Local dev runtime with WASI hosts |
| `examples/.env.example` | Environment variable template |
| `Cargo.toml` | Package manifest with dependencies |
| `.cargo/config.toml` | Credibil registry + credential providers |

## Configuration Files

Standard configuration files with templates in [configuration.md](configuration.md):

| File | Purpose |
| ---- | ------- |
| `.vscode/settings.json` | rust-analyzer wasm32 target configuration |
| `rustfmt.toml` | Nightly formatting rules |
| `rust-toolchain.toml` | Nightly channel + wasm32 target |
| `clippy.toml` | Lint exceptions (project-specific values) |
| `Makefile.toml` | CI/dev task runner (`cargo-make`) |
| `deny.toml` | Dependency license/advisory checks (`cargo-deny`) |
| `supply-chain/` | Supply-chain security audits (`cargo-vet`) |

GitHub Actions workflow files with templates in [configuration.md](configuration.md#github-workflows):

| File | Purpose |
| ---- | ------- |
| `.github/workflows/audit.yaml` | Daily scheduled security audit |
| `.github/workflows/ci.yaml` | CI on every push |
| `.github/workflows/patch.yaml` | Create patch release (manual) |
| `.github/workflows/publish.yaml` | CI, publish, and deploy pipeline (manual) |
| `.github/workflows/release.yaml` | Create release (manual) |

## File Ownership

| Scope | Responsibility |
| ----- | -------------- |
| `src/lib.rs` | Generated by guest-writer |
| `examples/*.rs` | Generated by guest-writer |
| `Cargo.toml` | Generated by guest-writer |
| `.github/workflows/*` | Generated by guest-writer |
| `crates/*` | Domain crates (external or from codegen) |
| Config files | Standard template files |
